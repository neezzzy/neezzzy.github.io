About Me

I like building tools that make tasks easier to understand, use, and improve.

Since discovering programming six years ago, I have kept coming back to projects that solve a clear workflow problem: study tools, dashboards, learning modules, and small web apps that help people get something done.

With a Master's Degree in Education and an Advanced Software Development certificate, I bring a mix of learning design and software development practice. That background helps me think about clear instructions, user flow, and documentation alongside the code.

My current work includes React, Svelte, JavaScript, TypeScript, Python, Node.js, PHP, Flask, MongoDB, and MySQL. I am still growing as a developer, with a focus on practical apps, automation, testing, and deployment workflows.

This portfolio highlights the kind of work I want to keep doing: useful interfaces, clear technical choices, and projects that are easy to explain to another person.

How I Build

A simple process for making useful things.

My best projects are practical and iterative: understand the task, make the first version work, then improve the details.

01

Understand the problem

Start with the workflow, user goal, and what would make the task easier.

02

Build a working version

Get the core path running first, then shape the interface around it.

03

Test and debug

Check behavior, edge cases, responsiveness, and anything that feels fragile.

04

Improve usability

Tighten labels, spacing, feedback states, and the steps a user repeats.

05

Document what changed

Leave notes, setup steps, and decisions clear enough for the next pass.
